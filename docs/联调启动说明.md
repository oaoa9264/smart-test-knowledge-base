# 联调启动说明

适用场景：本地同时启动后端（FastAPI）和前端（Vite）进行联调。

## 1. 首次启动前准备

### 后端依赖（仅首次）

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/backend
# 推荐 Python 3.8+ 虚拟环境
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -i https://pypi.org/simple --upgrade pip setuptools wheel
pip install -r requirements.txt
```

### 后端环境变量（建议）

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/backend
cp -n .env.example .env
```

注意：
- `backend/.env.example` 默认 `DATABASE_URL` 指向 Docker 内 PostgreSQL（`postgres:5432`）。
- 如果你是本机直启（未启动 Docker Postgres），请把 `DATABASE_URL` 改成 SQLite，例如：

```env
DATABASE_URL=sqlite:///./test_knowledge_base.db
```

如需启用 LLM 需求拆解，确认 `backend/.env` 至少包含：

```env
ANALYZER_PROVIDER=llm
AI_PARSE_PROVIDER=llm
OPENAI_API_KEY=your-openai-api-key-here
OPENAI_API_URL=https://api.openai.com/v1/chat/completions
OPENAI_TEXT_MODEL=gpt-4o
OPENAI_VISION_MODEL=gpt-4o
ZHIPU_API_KEY=your-zhipu-api-key-here
ZHIPU_API_URL=https://open.bigmodel.cn/api/paas/v4/chat/completions
LLM_THINKING_TYPE=disabled
```

说明：
- 当前 LLM 通道为 SSE（`httpx + text/event-stream`），不再依赖 `zai-sdk`。
- 多模型顺序为 `OpenAI -> 智谱`。若 OpenAI 调用失败会自动回退智谱；双模型都失败才由上层进入 `mock_fallback`。

### 前端依赖（仅首次）

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/frontend
npm install
```

## 2. 联调启动（两个命令行窗口）

### 窗口 A：启动后端

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/backend
[ -f .env ] && set -a && source .env && set +a
./.venv/bin/python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

后端地址：`http://localhost:8000`

### 窗口 B：启动前端

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/frontend
npm run dev -- --host 0.0.0.0 --port 5173
```

前端地址：`http://localhost:5173`

补充：
- 若页面不是在本机浏览器访问（例如手机/另一台电脑），前端默认 API 地址 `http://localhost:8000` 会指向“访问端本机”，需要在启动前设置：

```bash
export VITE_API_BASE_URL=http://<后端机器IP>:8000
```

## 3. 联调可用性检查

```bash
curl http://127.0.0.1:8000/health
```

预期返回：

```json
{ "status": "ok" }
```

### 3.1 需求拆解“分析引擎”标识检查

需求拆解页面会显示“分析引擎”标签（`LLM` / `Mock` / `Mock（LLM降级）`）。  
若显示“未知”或空白，通常是后端仍在运行旧代码，重启窗口 A 的后端进程后重试分析。

### 3.2 LLM 连通性快速自检（可选）

```bash
python -c "import socket; print(socket.gethostbyname('api.openai.com'))"
python -c "import socket; print(socket.gethostbyname('open.bigmodel.cn'))"
curl -I https://api.openai.com
curl -I https://open.bigmodel.cn
```

如果任一命令失败，通常是 DNS/网络策略问题，LLM 模式会降级为 `mock_fallback`。

## 4. 规则树异步生成联调验收

规则树会话式 AI 生成现已改为“异步启动 + 持久化阶段进度 + 刷新恢复”模式。

### 4.1 页面验收步骤

1. 进入“规则树”页面，打开“AI 规则树生成”抽屉。
2. 新建一个会话，输入需求文本，点击“开始生成”。
3. 预期：接口立即返回，页面进入阶段进度显示，而不是长时间等待一个同步响应。
4. 观察状态区阶段依次进入：
   - `生成规则树`
   - `AI 复核`
   - `保存结果`
   - `生成完成`
5. 在生成进行中直接刷新页面，再重新进入同一需求与同一会话。
6. 预期：页面能恢复当前阶段，并继续自动轮询直到完成。
7. 完成后，页面应恢复生成结果，并允许点击“应用到规则树”。
8. 点击“应用到规则树”后，顶部步骤条应切到“应用到规则树”，状态区显示“已应用到规则树 / 完成于 ...”。

### 4.2 后端重启中断验收

1. 重新发起一次规则树生成。
2. 在状态处于 `生成规则树 / AI 复核 / 保存结果` 期间重启后端进程。
3. 刷新页面后重新进入同一会话。
4. 预期：会话状态变为 `已中断`，提示类似“服务重启导致任务中断，请重新发起生成”。
5. 点击“重新生成”后，应能重新进入正常的异步生成流程。

### 4.3 接口语义说明

- `POST /api/rules/sessions/{session_id}/generate`
  - 现在只负责启动任务，不再同步返回完整规则树
  - 成功时返回 `{ accepted: true, session }`
- `GET /api/rules/sessions/{session_id}`
  - 返回会话当前状态、阶段文案、快照、错误信息和时间戳
  - 前端轮询与刷新恢复均基于该接口
- 后端启动时会自动把 `generating / reviewing / saving` 的历史会话统一改写为 `interrupted`

## 6. 风险收敛三阶段异步联调验收

“评审分析”“开发前分析”“提测前审计”现已改为“异步启动 + 持久化任务状态 + 刷新恢复”模式。

### 6.1 页面验收步骤

1. 进入“规则树”页面右侧风险面板。
2. 点击任一阶段按钮，例如“评审分析”。
3. 预期：接口立即返回，按钮不再长时间同步等待。
4. 按钮下方出现该阶段任务状态条，显示 `排队中 / 执行中 / 已完成 / 失败 / 已中断`。
5. 若该阶段之前已有成功结果，重新分析期间页面仍继续展示旧结果，不应闪空。
6. 在任务执行中直接刷新页面。
7. 预期：页面恢复阶段状态条；若该任务已有旧结果，刷新后仍能继续显示旧结果。
8. 任务完成后：
   - `review` / `pre_dev` 会刷新最新有效需求快照
   - 三个阶段都会刷新风险列表
   - 正式需求输入列表会同步刷新

### 6.2 后端重启中断验收

1. 重新发起一次任意阶段分析任务。
2. 在状态处于 `排队中 / 执行中` 时重启后端进程。
3. 刷新页面后回到风险面板。
4. 预期：对应阶段显示 `已中断`，并提示类似“服务重启导致任务中断，请重新发起分析”。

### 6.3 接口语义说明

- `POST /api/requirements/{requirement_id}/analysis-tasks/{stage}`
  - 只负责启动后台任务
  - 成功时返回 `{ accepted: true, task }`
- `GET /api/requirements/{requirement_id}/analysis-tasks/{stage}`
  - 返回该需求该阶段的最新任务
  - 用于单阶段轮询和刷新恢复
- `GET /api/requirements/{requirement_id}/analysis-tasks`
  - 一次性返回三个阶段的最新任务摘要
  - 用于页面初始化恢复
- 后端启动时会自动把 `queued / running` 的历史任务改写为 `interrupted`

## 7. 停止服务

- 在对应窗口按 `Ctrl + C`。
- 若进程在后台运行，可按端口结束：

```bash
for port in 8000 5173; do
  lsof -ti:$port | xargs kill 2>/dev/null || true
done
```
