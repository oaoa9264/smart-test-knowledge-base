# 联调启动说明

适用场景：本地同时启动后端（FastAPI）和前端（Vite）进行联调。

## 1. 首次启动前准备

### 后端依赖（仅首次）

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/backend
# 推荐 Python 3.8+ 虚拟环境
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -i https://pypi.org/simple --upgrade pip setuptools wheel
pip install -r requirements.txt
```

### 后端环境变量（建议）

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/backend
cp -n .env.example .env
```

如需启用 LLM 架构拆解，确认 `backend/.env` 至少包含：

```env
ANALYZER_PROVIDER=llm
ZHIPU_API_KEY=your-zhipu-api-key-here
ZHIPU_API_URL=https://open.bigmodel.cn/api/paas/v4/chat/completions
LLM_THINKING_TYPE=disabled
```

说明：当前 LLM 通道为 SSE（`httpx + text/event-stream`），不再依赖 `zai-sdk`。

### 前端依赖（仅首次）

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/frontend
npm install
```

## 2. 联调启动（两个命令行窗口）

### 窗口 A：启动后端

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/backend
source .venv/bin/activate
set -a && source .env && set +a
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

后端地址：`http://localhost:8000`

### 窗口 B：启动前端

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/frontend
npm run dev -- --host 0.0.0.0 --port 5173
```

前端地址：`http://localhost:5173`

## 3. 联调可用性检查

```bash
curl http://127.0.0.1:8000/health
```

预期返回：

```json
{ "status": "ok" }
```

### 3.1 架构拆解“分析引擎”标识检查

架构拆解页面会显示“分析引擎”标签（`LLM` / `Mock` / `Mock（LLM降级）`）。  
若显示“未知”或空白，通常是后端仍在运行旧代码，重启窗口 A 的后端进程后重试分析。

## 4. 在受限环境里“提权启动”说明（如 Codex 沙箱）

如果你是在受限沙箱里执行命令，端口绑定可能被拦截，需要允许提权后再执行同样命令。

后端启动命令（提权后执行）：

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/backend
source .venv/bin/activate
set -a && source .env && set +a
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

前端启动命令（提权后执行）：

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/frontend
npm run dev -- --host 0.0.0.0 --port 5173
```

## 5. 停止服务

- 在对应窗口按 `Ctrl + C`。
- 若进程在后台运行，可按端口结束：

```bash
lsof -ti:8000 | xargs kill -9
lsof -ti:5173 | xargs kill -9
```
