# 联调启动说明

适用场景：本地同时启动后端（FastAPI）和前端（Vite）进行联调。

## 1. 首次启动前准备

### 后端依赖（仅首次）

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/backend
# 推荐 Python 3.8+ 虚拟环境
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -i https://pypi.org/simple --upgrade pip setuptools wheel
pip install -r requirements.txt
```

### 后端环境变量（建议）

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/backend
cp -n .env.example .env
```

注意：
- `backend/.env.example` 默认 `DATABASE_URL` 指向 Docker 内 PostgreSQL（`postgres:5432`）。
- 如果你是本机直启（未启动 Docker Postgres），请把 `DATABASE_URL` 改成 SQLite，例如：

```env
DATABASE_URL=sqlite:///./test_knowledge_base.db
```

如需启用 LLM 需求拆解，确认 `backend/.env` 至少包含：

```env
ANALYZER_PROVIDER=llm
AI_PARSE_PROVIDER=llm
OPENAI_API_KEY=your-openai-api-key-here
OPENAI_API_URL=https://api.openai.com/v1/chat/completions
OPENAI_TEXT_MODEL=gpt-4o
OPENAI_VISION_MODEL=gpt-4o
ZHIPU_API_KEY=your-zhipu-api-key-here
ZHIPU_API_URL=https://open.bigmodel.cn/api/paas/v4/chat/completions
LLM_THINKING_TYPE=disabled
```

说明：
- 当前 LLM 通道为 SSE（`httpx + text/event-stream`），不再依赖 `zai-sdk`。
- 多模型顺序为 `OpenAI -> 智谱`。若 OpenAI 调用失败会自动回退智谱；双模型都失败才由上层进入 `mock_fallback`。

### 前端依赖（仅首次）

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/frontend
npm install
```

## 2. 联调启动（两个命令行窗口）

### 窗口 A：启动后端

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/backend
[ -f .env ] && set -a && source .env && set +a
./.venv/bin/python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

后端地址：`http://localhost:8000`

### 窗口 B：启动前端

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/frontend
npm run dev -- --host 0.0.0.0 --port 5173
```

前端地址：`http://localhost:5173`

补充：
- 若页面不是在本机浏览器访问（例如手机/另一台电脑），前端默认 API 地址 `http://localhost:8000` 会指向“访问端本机”，需要在启动前设置：

```bash
export VITE_API_BASE_URL=http://<后端机器IP>:8000
```

## 3. 联调可用性检查

```bash
curl http://127.0.0.1:8000/health
```

预期返回：

```json
{ "status": "ok" }
```

### 3.1 需求拆解“分析引擎”标识检查

需求拆解页面会显示“分析引擎”标签（`LLM` / `Mock` / `Mock（LLM降级）`）。  
若显示“未知”或空白，通常是后端仍在运行旧代码，重启窗口 A 的后端进程后重试分析。

### 3.2 LLM 连通性快速自检（可选）

```bash
python -c "import socket; print(socket.gethostbyname('api.openai.com'))"
python -c "import socket; print(socket.gethostbyname('open.bigmodel.cn'))"
curl -I https://api.openai.com
curl -I https://open.bigmodel.cn
```

如果任一命令失败，通常是 DNS/网络策略问题，LLM 模式会降级为 `mock_fallback`。

## 4. 在受限环境里“提权启动”说明（如 Codex 沙箱）

如果你是在受限沙箱里执行命令，端口绑定可能被拦截，需要允许提权后再执行同样命令。

后端启动命令（提权后执行）：

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/backend
[ -f .env ] && set -a && source .env && set +a
./.venv/bin/python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

前端启动命令（提权后执行）：

```bash
cd /Users/wanghu/Desktop/公司代码/smart-test-knowledge-base/frontend
npm run dev -- --host 0.0.0.0 --port 5173
```

## 5. 停止服务

- 在对应窗口按 `Ctrl + C`。
- 若进程在后台运行，可按端口结束：

```bash
for port in 8000 5173; do
  lsof -ti:$port | xargs kill 2>/dev/null || true
done
```
